## Top 10 des Vulnérabilités Web (OWASP)

### 1. Injection SQL

**Problème** : Manipulation de requêtes SQL via des inputs utilisateur

**Solution** :

```typescript
// ❌ VULNÉRABLE
const query = `SELECT * FROM users WHERE id = ${userId}`

// ✅ SÉCURISÉ - Requêtes paramétrées
const query = 'SELECT * FROM users WHERE id = ?'
db.query(query, [userId])

// Avec ORM (Prisma)
const user = await prisma.user.findUnique({
  where: { id: userId },
})
```

### 2. Authentification Cassée

Implémentation sécurisée avec JWT :

```typescript
import bcrypt from 'bcrypt'
import jwt from 'jsonwebtoken'

// Hashage du mot de passe
const hashedPassword = await bcrypt.hash(password, 12)

// Création du token
const token = jwt.sign({ userId: user.id, email: user.email }, process.env.JWT_SECRET, {
  expiresIn: '1h',
})

// Vérification
const decoded = jwt.verify(token, process.env.JWT_SECRET)
```

### 3. XSS (Cross-Site Scripting)

Protection contre XSS :

```typescript
// Sanitization des inputs
import DOMPurify from 'isomorphic-dompurify'

const cleanHTML = DOMPurify.sanitize(userInput)
```

## Chiffrement

### HTTPS Everywhere

Configuration Nginx :

```nginx
server {
    listen 443 ssl http2;
    server_name systech.com;

    ssl_certificate /etc/ssl/certs/systech.crt;
    ssl_certificate_key /etc/ssl/private/systech.key;

    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;
    ssl_prefer_server_ciphers on;

    add_header Strict-Transport-Security "max-age=31536000; includeSubDomains" always;
}
```

### Chiffrement des Données

```typescript
import crypto from 'crypto'

// Chiffrement AES-256
const algorithm = 'aes-256-gcm'
const key = crypto.randomBytes(32)
const iv = crypto.randomBytes(16)

function encrypt(text: string): string {
  const cipher = crypto.createCipheriv(algorithm, key, iv)
  let encrypted = cipher.update(text, 'utf8', 'hex')
  encrypted += cipher.final('hex')
  const authTag = cipher.getAuthTag()

  return encrypted + ':' + authTag.toString('hex')
}

function decrypt(encrypted: string): string {
  const parts = encrypted.split(':')
  const decipher = crypto.createDecipheriv(algorithm, key, iv)
  decipher.setAuthTag(Buffer.from(parts[1], 'hex'))

  let decrypted = decipher.update(parts[0], 'hex', 'utf8')
  decrypted += decipher.final('utf8')

  return decrypted
}
```

## Rate Limiting

Prévention des attaques par force brute :

```typescript
import rateLimit from 'express-rate-limit'

const loginLimiter = rateLimit({
  windowMs: 15 * 60 * 1000, // 15 minutes
  max: 5, // 5 tentatives
  message: 'Trop de tentatives, réessayez plus tard',
  standardHeaders: true,
  legacyHeaders: false,
})

app.post('/api/login', loginLimiter, loginController)
```

## Validation des Entrées

Validation robuste avec Zod :

```typescript
import { z } from 'zod'

const userSchema = z.object({
  email: z.string().email(),
  password: z
    .string()
    .min(12, 'Minimum 12 caractères')
    .regex(/[A-Z]/, 'Au moins une majuscule')
    .regex(/[a-z]/, 'Au moins une minuscule')
    .regex(/[0-9]/, 'Au moins un chiffre')
    .regex(/[^A-Za-z0-9]/, 'Au moins un caractère spécial'),
  age: z.number().int().min(18).max(120),
})

try {
  const validatedData = userSchema.parse(userData)
  // Traitement sécurisé
} catch (error) {
  // Erreur de validation
}
```

## Headers de Sécurité

Configuration avec Helmet.js :

```typescript
import helmet from 'helmet'

app.use(
  helmet({
    contentSecurityPolicy: {
      directives: {
        defaultSrc: ["'self'"],
        styleSrc: ["'self'", "'unsafe-inline'"],
        scriptSrc: ["'self'"],
        imgSrc: ["'self'", 'data:', 'https:'],
      },
    },
    hsts: {
      maxAge: 31536000,
      includeSubDomains: true,
      preload: true,
    },
    frameguard: {
      action: 'deny',
    },
    noSniff: true,
    xssFilter: true,
  })
)
```
